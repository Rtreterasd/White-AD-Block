WhiteAdBlock

Privacy Policy

Last updated: April 15, 2026. This page describes how WhiteAdBlock handles user data and local browser settings.

Overview

WhiteAdBlock is a browser extension designed to block ads, trackers, consent popups and unwanted visual elements directly in the browser. The extension is designed to process its functionality locally on the user's device.

Data handling

WhiteAdBlock does not sell user data and does not transfer personal data to third parties for advertising, profiling or analytics.

The extension analyzes visited pages locally in order to detect and hide advertising, tracking and consent-related elements. Blocking rules and counters are processed locally in the browser.

Data the extension may process locally

  • visited website domains and page context needed to apply rules
  • website content structure required to identify ads and overlays
  • locally stored user settings and custom blocking rules
  • aggregate blocked counters used in the extension interface

Data the extension does not intentionally collect

  • personally identifiable information
  • financial or payment information
  • authentication credentials such as passwords or PIN codes
  • medical information
  • private messages, emails or chat contents
  • GPS location or precise location data

Local storage

The extension may store the following data locally in browser storage:

  • enabled or disabled protection state
  • user-created site rules and global rules
  • aggregate blocked counters
  • internal feature state required for stable extension behavior

Remote code

WhiteAdBlock does not use remote code. All executable JavaScript, CSS and bundled assets are shipped inside the extension package.

External links, if present, are standard user-initiated links and are not used to download and execute remote code inside the extension.
